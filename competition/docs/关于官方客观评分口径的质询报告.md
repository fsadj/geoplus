## 二、这条回复解决了什么

这条回复实际上解决了此前最核心的两个口径歧义：

1. `|s|` 的基本长度单位，不应再按“分词后的词数”理解，而应按中文语境下的字符数理解。
2. 三项客观分都不是绝对累加值，而是“目标来源在当前回答所有被引用得分中的份额”。

因此，三项客观分可以统一理解为：

- `word_volu`：目标来源拿走了多少“被引用字符份额”。
- `posi_prom`：目标来源拿走了多少“被引用位置权重份额”。
- `word_posi`：目标来源拿走了多少“被引用加权字符份额”。

这意味着，优化目标不再是单纯“让文本更长”，而是“让回答中更多高权重引用句优先落到自己身上”。

## 三、本地 simulator 已做的对应修改

本地实现已经按这条回复完成调整：

1. `word_volu` 改为按 `target_chars / total_cited_chars` 计算。
2. `posi_prom` 改为按 `target_position_weight / total_cited_position_weight` 计算。

## 五、对优化策略的直接影响

这条回复对内容优化的影响非常明确：

1. “继续扩写”不再是默认正确方向。长文本只有在能抢到更多被引用份额时才有价值。
2. 重点应从“文长”转向“高密度可摘取句”。需要让模型更愿意直接抽走我们的判断句、定义句、对比句和结论句。
3. 最有价值的提升来自三件事：更早被引用、单次被引用片段更长、在同类论点上替代其他来源。
4. 过长但松散的铺陈会把回答的总引用池做大，反而可能稀释自己的份额。

从策略语言上说，未来应优先追求：

- 结论先行
- 争议澄清
- 对比式判断
- 可直接摘抄的定义句
- 多维度但收束清晰的论证骨架

## 八、当前最可执行的结论

如果现在就要继续优化，我会按以下优先级推进：

1. **主路线优先使用 `rebuttal` 结构。** 它在主内容对比中拿到最高平均增量。
2. **长度优先落在 `extended` 或 `compact` 带宽，而不是 `ultra`。** 其中 `extended` 更强，`compact` 更稳。
3. **不要再把“补充更多材料”本身视为优化目标。** 真正的目标是增加高权重可摘取句的份额。
4. **后续新模板应围绕“高密度反驳 + 可引用结论句 + 争议澄清”继续扩样。**

换句话说，当前最优方向不是“写得最大”，而是“写得更像一组会被模型直接抽走的前排判断句”。

截至目前，最稳妥的整体判断是：
